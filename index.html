Trust foundation

Security and recoverability built into the foundation

EasyHisab is built around authenticated access, business-level access boundaries and recoverability practices that matter for financial records.

Google identity Sign in through Google; there is no EasyHisab password login flow.
JWT access control API routes require signed tokens with issuer and audience validation.
Refresh-token hygiene Refresh tokens are stored hashed server-side and rotated when sessions refresh.
Business-level access Business-scoped APIs verify that the signed-in user can access that business.
Backup runbook Deployment includes PostgreSQL backup scripts with configurable interval and retention.
Privacy controls Users can control diagnostics preferences from the app settings screen.
